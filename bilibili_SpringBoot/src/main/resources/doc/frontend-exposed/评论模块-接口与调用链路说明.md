# 评论模块接口与调用链路说明（CommentController / MeCommentController）

## 1. 范围

本文覆盖以下两个控制器，并沿着“控制层 -> 服务层 -> 数据访问层”说明每个接口的处理链路：

- `com.bilibili.controller.CommentController`
- `com.bilibili.controller.MeCommentController`

## 2. 模块总览

| 控制器 | 路径前缀 | 主要职责 |
| --- | --- | --- |
| `CommentController` | `/videos` | 视频评论查询（公开） |
| `MeCommentController` | `/me` | 当前登录用户发表评论、删除评论、评论点赞/取消点赞 |

### 2.1 控制层到服务层映射

| 控制器方法 | Service 方法 |
| --- | --- |
| `CommentController.listComments` | `CommentService.listComments(Long videoId, PageQueryDTO pageQuery, Long currentUid)` |
| `MeCommentController.createComment` | `CommentService.createComment(Long uid, Long videoId, CommentCreateDTO dto)` |
| `MeCommentController.deleteComment` | `CommentService.deleteComment(Long uid, Long commentId)` |
| `MeCommentController.likeComment` | `CommentService.likeComment(Long uid, Long commentId)` |
| `MeCommentController.unlikeComment` | `CommentService.unlikeComment(Long uid, Long commentId)` |

## 3. 接口明细

## 3.1 `GET /videos/{videoId}/comments`

- 控制器：`CommentController.listComments`
- 鉴权：公开接口（无需登录）
- 路径参数：`videoId`
- 查询参数：`pageNo`、`pageSize`（默认 `1` / `10`，`pageSize` 最大 `50`）
- 返回：`Result<List<CommentVO>>`

### 返回结构特点

每个根评论（`parentId=0`）包含 `childComments` 子数组，子数组内是该根评论的一层回复。

### 调用链路

1. `CommentController.listComments(videoId, pageQuery, currentUser)` 获取当前登录用户 uid（未登录则为 `null`）。
2. 调用 `commentService.listComments(videoId, pageQuery, currentUid)`。
3. `CommentServiceImpl.listComments` 先校验 `videoId`，并检查视频是否存在且状态正常。
4. 按分页查询当前页根评论：`video_id + status=0 + parent_id=0`，按 `create_time desc, id desc`。
5. 取根评论 id 列表，查询回复评论：`video_id + status=0 + root_id in (...) + parent_id != 0`，按 `create_time asc, id asc`。
6. 批量查询评论作者 `t_user_info`，补齐 `nickname/avatar`。
7. 若当前用户已登录，批量查询其点赞关系（`t_comment_like`）并计算每条评论的 `isLiked`。
8. 组装树形结果：根评论在外层列表，回复放入各自 `childComments`。

### 涉及数据

- 表：`t_video`（存在性校验）
- 表：`t_comment`（根评论 + 回复查询）
- 表：`t_user_info`（评论用户信息）
- 表：`t_comment_like`（当前用户点赞态查询）

## 3.2 `POST /me/videos/{videoId}/comments`

- 控制器：`MeCommentController.createComment`
- 鉴权：类级 `@PreAuthorize("isAuthenticated()")`
- 路径参数：`videoId`
- 请求体：`CommentCreateDTO`

```json
{
  "content": "这视频讲得很清楚",
  "parentId": 0
}
```

- 成功返回：`Result<Long>`（`data` 为新评论 id）

### 调用链路

1. JWT 过滤器解析 token，控制器通过 `@AuthenticationPrincipal` 获取当前 uid。
2. `MeCommentController.createComment` 调用 `commentService.createComment(uid, videoId, dto)`。
3. `CommentServiceImpl.createComment` 进行参数校验：`uid/videoId`、请求体、`content` 非空、长度不超过 `1000`。
4. 校验目标视频存在且状态正常。
5. 处理层级：
   1. `parentId` 为空按 `0` 处理，表示根评论。
   2. `parentId > 0` 时校验父评论存在、属于同一视频、且父评论必须是根评论（只支持一级回复）。
6. 插入 `t_comment`（`status=0`，初始化 `like_count=0`、`reply_count=0`）。
7. 更新计数：
   1. 视频 `comment_count + 1`。
   2. 若为回复，父评论 `reply_count + 1`。
8. 返回新评论 id。

### 涉及数据

- 表：`t_video`（存在性校验、`comment_count` 更新）
- 表：`t_comment`（插入、父评论校验、`reply_count` 更新）
- 事务：`@Transactional(rollbackFor = Exception.class)`

## 3.3 `DELETE /me/comments/{commentId}`

- 控制器：`MeCommentController.deleteComment`
- 鉴权：
  - 类级 `@PreAuthorize("isAuthenticated()")`
  - 方法级 `@PreAuthorize("@authz.canDeleteComment(authentication, #commentId)")`
- 路径参数：`commentId`
- 返回：`Result<Void>`

### 调用链路

1. 进入方法前，`AuthzService.canDeleteComment` 先校验当前用户是否是该评论作者（且评论状态正常）。
2. 控制器调用 `commentService.deleteComment(uid, commentId)`。
3. `CommentServiceImpl.deleteComment` 二次校验评论存在与作者归属，不满足则报错。
4. 软删除当前评论：`t_comment.status: 0 -> 1`。
5. 分支处理：
   1. 若删除的是回复评论（`parentId > 0`），父评论 `reply_count - 1`（最低到 0）。
   2. 若删除的是根评论，同时软删除该根下所有正常回复（`root_id = commentId`）。
6. 按实际删除条数回写视频 `comment_count`（最低到 0）。

### 涉及数据

- 表：`t_comment`（软删除、父评论回复数更新）
- 表：`t_video`（`comment_count` 递减）
- 事务：`@Transactional(rollbackFor = Exception.class)`

## 3.4 `POST /me/comments/{commentId}/likes`

- 控制器：`MeCommentController.likeComment`
- 鉴权：类级 `@PreAuthorize("isAuthenticated()")`
- 路径参数：`commentId`
- 返回：`Result<Void>`

### 调用链路

1. 控制器拿到当前 uid 后调用 `commentService.likeComment(uid, commentId)`。
2. `CommentServiceImpl.likeComment` 校验参数并确认评论存在。
3. 查询点赞关系 `t_comment_like(user_id, comment_id)`：
   1. 不存在：插入新关系（`status=0`）。
   2. 存在且已点赞（`status=0`）：直接返回（幂等）。
   3. 存在但为取消状态（`status=1`）：更新为 `status=0`。
4. 点赞生效时，评论 `like_count + 1`。

### 涉及数据

- 表：`t_comment_like`（插入或更新）
- 表：`t_comment`（`like_count` 更新）
- 事务：`@Transactional(rollbackFor = Exception.class)`

## 3.5 `DELETE /me/comments/{commentId}/likes`

- 控制器：`MeCommentController.unlikeComment`
- 鉴权：类级 `@PreAuthorize("isAuthenticated()")`
- 路径参数：`commentId`
- 返回：`Result<Void>`

### 调用链路

1. `MeCommentController.unlikeComment` 调 `commentService.unlikeComment(uid, commentId)`。
2. `CommentServiceImpl.unlikeComment` 校验参数并确认评论存在。
3. 仅将“当前处于点赞状态”的关系改为取消状态：`t_comment_like.status: 0 -> 1`。
4. 若更新行为 `0` 行，直接返回（幂等，表示原本未点赞）。
5. 更新成功时，评论 `like_count - 1`（最低到 0）。

### 涉及数据

- 表：`t_comment_like`（软取消）
- 表：`t_comment`（`like_count` 递减）
- 事务：`@Transactional(rollbackFor = Exception.class)`

## 4. 鉴权与错误码约定

## 4.1 鉴权入口

- `/videos/{videoId}/comments`：公开接口。
- `/me/videos/{videoId}/comments`、`/me/comments/{commentId}`、`/me/comments/{commentId}/likes`：需要登录（JWT）。
- 删除评论额外需要满足资源权限：必须是评论作者（`AuthzService.canDeleteComment`）。

## 4.2 统一返回结构

所有接口返回 `Result<T>`：

- 成功：`{"code":0,"message":"OK","data":...}`
- 失败：`{"code":错误码,"message":"错误信息","data":null}`

## 4.3 常见错误

- 400：参数/业务错误（如 `videoId` 无效、`comment not found`、`content is too long`、`only one-level reply is supported`）
- 401：未登录访问 `/me/**`
- 403：已登录但无权限（如删除他人评论）
- 500：内部异常（如计数字段更新失败）
