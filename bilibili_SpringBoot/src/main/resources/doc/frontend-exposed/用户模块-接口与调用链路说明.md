# 用户模块接口与调用链路说明（UserController / MeUserController）

## 1. 范围

本文只覆盖以下两个控制器，并沿着“控制层 -> 服务层 -> 数据访问层”说明每个接口的处理链路：

- `com.bilibili.controller.UserController`
- `com.bilibili.controller.MeUserController`

## 2. 模块总览

| 控制器 | 路径前缀 | 主要职责 |
| --- | --- | --- |
| `UserController` | `/users` | 注册、登录、公开资料查询 |
| `MeUserController` | `/me` | 当前登录用户的资料修改 |

### 2.1 控制层到服务层映射

| 控制器方法 | Service 方法 |
| --- | --- |
| `UserController.register` | `UserService.register(UserRegisterDTO)` |
| `UserController.login` | `UserService.login(UserLoginDTO)` + `JwtTokenService.generateToken` |
| `UserController.getPublicProfile` | `UserService.getPublicProfile(Long uid)` |
| `MeUserController.updateMyProfile` | `UserService.updatePublicProfile(Long uid, UserProfileUpdateDTO)` |

## 3. 接口明细

## 3.1 `POST /users/register`

- 控制器：`UserController.register`
- 鉴权：公开接口（无需登录）
- 请求体：`UserRegisterDTO`

```json
{
  "username": "u1001",
  "nickname": "Alice",
  "password": "123456",
  "confirmPassword": "123456"
}
```

- 成功返回：`Result<Long>`（`data` 为新用户 uid）

### 调用链路

1. `UserController.register(dto)` 调用 `userService.register(dto)`。
2. `UserServiceImpl.register` 进行参数校验：空值校验、两次密码一致性校验。
3. 通过 `UserMapper.selectOne` 按用户名查重。
4. 插入 `t_user`（用户名、加密密码、状态），拿到用户主键。
5. 插入 `t_user_info`（昵称、关注数/粉丝数初始化）。
6. 返回新用户 uid。

### 涉及数据

- 表：`t_user`、`t_user_info`
- 事务：`@Transactional(rollbackFor = Exception.class)`（两表写入同事务）

## 3.2 `POST /users/login`

- 控制器：`UserController.login`
- 鉴权：公开接口（无需登录）
- 请求体：`UserLoginDTO`

```json
{
  "username": "u1001",
  "password": "123456"
}
```

- 成功返回：`Result<UserLoginVO>`，其中 `data.token` 为 JWT。

### 调用链路

1. `UserController.login(dto)` 调用 `userService.login(dto)`。
2. `UserServiceImpl.login` 校验参数并对密码做 SHA-256 计算。
3. `UserMapper.selectOne` 按 `username + passwordHash` 查询。
4. 返回 `UserLoginVO(uid, username)`。
5. 控制器调用 `JwtTokenService.generateToken(uid)` 生成 token，并写入 `loginVO.token`。
6. 返回前端。

### 涉及数据

- 表：`t_user`（只读）
- 组件：`JwtTokenService`

## 3.3 `GET /users/{uid}`

- 控制器：`UserController.getPublicProfile`
- 鉴权：公开接口（无需登录）
- 路径参数：`uid`
- 返回：`Result<UserProfileVO>`

### 调用链路

1. `UserController.getPublicProfile(uid)` 调用 `userService.getPublicProfile(uid)`。
2. `UserServiceImpl.getPublicProfile` 校验 uid。
3. 通过 `UserInfoMapper.selectOne` 查询 `t_user_info`。
4. 映射为 `UserProfileVO` 返回。

### 涉及数据

- 表：`t_user_info`（只读）

## 3.4 `PUT /me/profile`

- 控制器：`MeUserController.updateMyProfile`
- 鉴权：类级 `@PreAuthorize("isAuthenticated()")`
- 请求体：`UserProfileUpdateDTO`

```json
{
  "nickname": "AliceNew",
  "sign": "hello"
}
```

- 返回：`Result<Void>`

### 调用链路

1. JWT 过滤器从 `Authorization: Bearer <token>` 解析出 `AuthenticatedUser`。
2. 控制器通过 `@AuthenticationPrincipal` 拿到 `currentUser.uid`。
3. 调用 `userService.updatePublicProfile(uid, dto)`。
4. `UserServiceImpl.updatePublicProfile` 做 uid 与更新内容校验。
5. `UserInfoMapper.update` 按 `user_id` 执行选择性更新（昵称/签名）。

### 涉及数据

- 表：`t_user_info`（更新）

## 4. 鉴权与错误码约定

## 4.1 鉴权入口

- `/users/login`、`/users/register`、`/users/{uid}`：公开。
- `/me/**`：需要登录（JWT）。

## 4.2 统一返回结构

所有接口返回 `Result<T>`：

- 成功：`{"code":0,"message":"OK","data":...}`
- 失败：`{"code":错误码,"message":"错误信息","data":null}`

## 4.3 常见错误

- 400：业务参数错误（`IllegalArgumentException`）
- 401：未认证（`login required`）
- 403：已认证但无权限（`access denied`）
- 500：服务器内部异常
