# 用户行为模拟造数说明（纯 API 版）

## 1. 目标
- 只通过后端接口造数据，不直接插入 SQL。
- 保证表间数据一致性（视频、点赞、评论、关注等关系由业务接口维护）。

## 2. 两个脚本职责
- `tools/simulator/seed_baseline_data.py`
  - 通过 `/users/register`、`/users/login` 创建用户并生成账号密码本。
  - 通过上传接口 `/me/videos/uploads/*` 创建基座视频。
- `tools/simulator/simulate_user_behavior.py`
  - 登录用户后并发执行行为：
  - 列表、详情、播放、点赞/取消、评论、关注/取关、改资料、传头像、上传视频。

## 3. 基座脚本（纯 API）
```bash
python tools/simulator/seed_baseline_data.py \
  --base-url http://localhost:8080 \
  --user-count 1000 \
  --video-count 500 \
  --workers 50 \
  --video-workers 24
```

说明：
- `--workers`：用户注册/登录并发。
- `--video-workers`：视频上传并发。
- `--video-count`：接口创建视频数量（不是 SQL 插入）。

## 4. 行为脚本（纯 API）
```bash
python tools/simulator/simulate_user_behavior.py \
  --base-url http://localhost:8080 \
  --accounts-file tools/simulator/output/user_accounts.csv \
  --duration-seconds 1800 \
  --interval-ms 50 \
  --login-workers 50 \
  --workers 32
```

## 5. 输出文件
- 用户台账：
  - `tools/simulator/output/user_accounts.csv`
  - `tools/simulator/output/user_accounts.json`
- 视频台账（接口创建）：
  - `tools/simulator/output/seed_video_ids.csv`
  - `tools/simulator/output/seed_video_ids.json`

## 6. 注意
- 该方案不再生成 `seed_videos.sql`，也不再依赖 SQL 兜底读取视频 ID。
- 如果 `/videos` 接口异常，行为脚本会直接失败并提示先修接口，而不是再走本地 SQL。
