## 核心层级说明

### 1. Controller 层 (RESTful API)
- 遵循 RESTful 规约：`GET /videos/{id}`，`POST /videos`。
- 负责接口鉴权（通过 Security 注解）、参数校验、结果包装。

### 2. Service 层
- 包含 `IService` 接口与 `ServiceImpl` 实现类。
- **事务控制**：在实现类上使用 `@Transactional`。
- **异步处理**：投稿、发送通知等耗时任务使用 `@Async` 配合线程池。

### 3. Mapper 层 (Persistence)
- 继承 MyBatis-Plus 的 `BaseMapper<T>`。
- 复杂的统计查询、多表联查放置在 `src/main/resources/mapper/` 的 XML 中。

### 4. Model 层 (规约命名)
- **DO (Data Object)**：与数据库表结构一一对应（如 `UserDO`, `VideoDO`）。
- **DTO (Data Transfer Object)**：用于接收前端各种 Body 参数（如 `UserRegisterDTO`）。
- **VO (View Object)**：经过脱敏或数据聚合后的显示对象（如 `VideoDetailVO`）。

### 5. Redis 实现
- **排行榜**：使用 `ZSET` 结构（Key: `rank:video:click`）。
- **搜索记录**：使用 `LIST` 结构（Key: `search:history:{userId}`）。

### 6. 配置中心 (Config)
- 采用 Spring 官方推荐的 **Java Config** 代替沉重的 XML 配置（除了必选的 `web.xml`）。
