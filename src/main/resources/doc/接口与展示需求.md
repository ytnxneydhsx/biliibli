# 接口与展示需求 (BFF 架构 - 动静分离版)

本文档定义了前端展示层所需的数据结构 (VO)。后端在实现时必须基于这些 **View Object** 进行查询组装。

> V1 对齐口径：以 `bilibili.sql` 为准，先完成 `Spring + Spring MVC + MyBatis-Plus + MySQL` 主链路。  
> Redis/安全框架等能力属于 V2 增强，不作为 V1 必选依赖。

## 核心设计理念：动静分离
- **静态数据 (Static/Public)**：所有用户看到的都一样，基本不改。**后端应积极缓存** (标题、封面、视频地址、作者资料)。
- **动态数据 (Dynamic/Fast)**：实时变化，不能长缓存 (播放量、点赞数)。
- **个性化数据 (User-Specific)**：因人而异 ( "我"是否点赞、"我"是否关注 )，必须实时查/依赖 Session。

## 1. 首页/列表流 (VideoCardVO)
**前端组件**：`VideoCard` (轻量级)
**场景**：首页推荐、搜索结果、分区列表。

| 字段名 (Field)   | 类型   | 属性 | 说明                          | 来源                   |
| :--------------- | :----- | :--- | :---------------------------- | :--------------------- |
| **id**           | Long   | 🗝️    | 视频 ID                       | `t_video.id`           |
| **title**        | String | 🟢 静 | 视频标题                      | `t_video.title`        |
| **coverUrl**     | String | 🟢 静 | 封面图 URL                    | `t_video.cover_url`    |
| **duration**     | String | 🟢 静 | 时长 (格式化后 "04:20")       | `t_video.duration`     |
| **authorName**   | String | 🟢 静 | UP主昵称                      | `t_user_info.nickname` |
| **createTime**   | String | 🟢 静 | 发布时间 (相对时间 "2小时前") | `t_video.create_time`  |
| **viewCount**    | String | 🟠 动 | 播放量 (格式化 "1.2万")       | `t_video.view_count`   |
| **danmakuCount** | String | 🟠 动 | 弹幕概览数 (可选)             | (关联查询或统计表)     |

## 2. 视频详情页 (VideoDetailVO) 
**前端组件**：`VideoPlayer`, `VideoInfoBar`, `AuthorCard`
**场景**：核心播放页。此 VO 包含**静态信息**与**用户交互状态**两部分。

### Part A: 静态信息 (VideoStaticInfo)
*建议缓存*

| 字段名         | 类型         | 说明                             | 来源                     |
| :------------- | :----------- | :------------------------------- | :----------------------- |
| **id**         | Long         | 视频 ID                          | `t_video.id`             |
| **videoUrl**   | String       | **视频源地址** (MP4/M3U8)        | `t_video.video_url`      |
| **title**      | String       | 视频标题                         | `t_video.title`          |
| **desc**       | String       | 视频简介                         | `t_video.description`    |
| **uploadDate** | Schema       | 发布日期 (2023-10-01 12:00)      | `t_video.create_time`    |
| **tags**       | List<String> | 标签列表                         | `t_video_tag` -> `t_tag` |
| **author**     | UserTinyVO   | 作者的小卡片 (头像/昵称/ID/签名) | `t_user_info`            |

### Part B: 动态与交互 (VideoUserInteraction)
*实时查询，依赖当前登录用户 (currentUserId)*

| 字段名         | 类型    | 说明               | 来源                 |
| :------------- | :------ | :----------------- | :------------------- |
| **viewCount**  | Long    | 实时播放量         | `t_video.view_count` |
| **likeCount**  | Long    | 实时点赞数         | `t_video.like_count` |
| **coinCount**  | Long    | 硬币数 (预留)      | (V2 预留，V1 可不返回) |
| **isLiked**    | Boolean | **我是否点赞**     | 查询 `t_video_like`  |
| **isFollowed** | Boolean | **我是否关注作者** | 查询 `t_following`   |

## 3. 评论区 (CommentVO)
**前端组件**：`CommentSection`
**场景**：视频下方的交互区域。

| 字段名            | 类型       | 属性   | 说明                   | 来源                    |
| :---------------- | :--------- | :----- | :--------------------- | :---------------------- |
| **id**            | Long       | 🗝️      | 评论 ID                | `t_comment.id`          |
| **content**       | String     | 🟢 静   | 评论内容               | `t_comment.content`     |
| **createTime**    | Schema     | 🟢 静   | 发布时间               | `t_comment.create_time` |
| **userInfo**      | UserTinyVO | 🟢 静   | 发送者信息 (头像/昵称) | `t_user_info`           |
